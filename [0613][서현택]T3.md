# T3 Sanity-Test— Leakage (A vs B)


## 1. 실험 개요

| 항목 | 내용 |
|------|------|
| 목적 | GT 헤어 외형 누설(leakage) 의존도 측정 (A vs B 단일 변수) |
| 비교 모델 | Ours (mcs1) vs Sketch-only (mcs3) |
| 입력 고정 | GT 스케치 + GT matte (전 조건 동일) |
| 변수 | 타겟 이미지: A(원본 GT 헤어) vs B(HairMapper bald) |
| 이미지 | CM_1051 (unbraid, 1장) + braid_2562 (braid, 1장) — 1차 방향 확인용 |
| seed / steps | 고정 |

**A**: 원본 이미지 그대로 (GT 헤어 + 원본 장면)  
**B**: HairMapper로 헤어 제거 (bald + 원본 장면, 장면·조명 유지)


## 2.. Figure

### 모델 정의

| 명칭 | 내부 코드 | MatteCNN | matte_raw | gate | ControlNet 입력 (17ch) |
|------|-----------|:---:|:---:|:---:|---|
| **Ours** | mcs1 | ✅ ON | ✅ ON | ❌ OFF | `cat([sketch_lat + MatteCNN_feat, matte_raw])` — 스케치 + MatteCNN 헤어 기하 feature + raw matte 1ch |
| **Sketch-only** | mcs3 | ❌ OFF | ❌ OFF | ❌ OFF | `cat([sketch_lat + zeros, zeros])` — 스케치만, matte 채널 전부 0으로 zeroed |

> **핵심 차이**: Ours는 MatteCNN이 matte를 인코딩해 스케치 latent에 더하고, raw matte 1ch도 ControlNet에 별도 공급. Sketch-only는 동일한 17ch 구조지만 MatteCNN·matte_raw 채널을 모두 0으로 강제 → 순수 스케치 신호만 작동.
